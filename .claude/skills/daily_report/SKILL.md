# 日報自動生成

今日または指定日の作業内容を分析し、構造化された日報を生成します。

## 生成手順

### 1. 情報収集

以下の情報を収集します:

**gitコミット履歴**（指定日または昨日）:
```bash
# 引数から日付を取得（YYYYMMDD形式またはYYYY-MM-DD形式を両方サポート）
ARG="$1"
if [ -z "$ARG" ]; then
    TARGET_DATE=$(date -v-1d +%Y-%m-%d)
elif echo "$ARG" | grep -qE '^[0-9]{8}$'; then
    # YYYYMMDDをYYYY-MM-DDに変換
    TARGET_DATE="${ARG:0:4}-${ARG:4:2}-${ARG:6:2}"
else
    TARGET_DATE="$ARG"
fi
FORMATTED_DATE=$(echo $TARGET_DATE | sed 's/-//g')
YEAR=$(echo $TARGET_DATE | cut -c1-4)
MONTH=$(echo $TARGET_DATE | cut -c6-7)

git log --since="${TARGET_DATE} 00:00:00" --until="${TARGET_DATE} 23:59:59" --pretty=format:"%h|%ai|%s" --reverse
git log --since="${TARGET_DATE} 00:00:00" --until="${TARGET_DATE} 23:59:59" --stat
```

**ファイル変更履歴**:
```bash
git log --since="${TARGET_DATE} 00:00:00" --until="${TARGET_DATE} 23:59:59" --name-status --pretty=format:
```

**プロジェクト情報**:
```bash
basename $(git rev-parse --show-toplevel 2>/dev/null || echo "unknown")
git branch --show-current
```

**会話履歴**: この会話でのやり取り、実施した作業、設計判断を自動的に参照

### 2. 出力ファイル構成

3種類のファイルを生成します:

```
BASE=/Users/sukenobu/tfile/00.memo/kintai/${YEAR}/${MONTH}

${BASE}/detail/${FORMATTED_DATE}.md   # 詳細日報
${BASE}/week/${YEAR}${MONTH}.md       # 週次サマリー(月ごとに1ファイル, 追記形式)
${BASE}/month/${YEAR}${MONTH}.md      # 月次一覧(月ごとに1ファイル, 追記形式)
```

week・monthともにファイル名は `YYYYMM.md`(例: `202602.md`)を使用します。

### 3. 各フォーマット

#### detail: 詳細日報
```markdown
# YYYY-MM-DD 作業内容

## プロジェクト
[プロジェクト名]

## 実施内容

### 1. [作業タイトル] (HH:MM)
**コミット**: [hash] "[メッセージ]"

**修正ファイル**: `[ファイル名]`

**修正内容**:
- [詳細な説明]
- [変更の目的や背景]

**背景**: [なぜこの作業が必要だったか]

---

### 2. [次の作業タイトル] (HH:MM)
...

---

## 成果

### [カテゴリ1（例: データ品質向上）]
- [具体的な成果1]
- [具体的な成果2]

### [カテゴリ2（例: 運用改善）]
- [具体的な成果3]

## ブランチ
`[ブランチ名]`

## 次回作業予定
- [TODO1]
- [TODO2]
```

#### week: 週次サマリー（約5行、追記形式）
既存の週次ファイルに以下の形式で1日分のエントリを追記します:
```markdown
# YYYYMMDD
## 作業内容
- [プロジェクト/機能名]
  - [作業カテゴリ]
    - [具体的な作業1（30字以内）]
    - [具体的な作業2（30字以内）]
```

1エントリ5行程度を目安にし、箇条書きは階層を3段階まで使い簡潔に記述します。
ファイルが既に存在する場合は末尾に追記します。存在しない場合は新規作成します。

#### month: 月次一覧（1行、追記形式）
既存の月次ファイルに以下の形式で1行追記します:
```
# YYYYMMDD プロジェクト名(主な作業1, 主な作業2)
```

カッコ内の各作業は10〜15字程度の短い語句にまとめます。
ファイルが既に存在する場合は末尾に追記します。存在しない場合は新規作成します。

### 4. 保存

ディレクトリを作成してから各ファイルに保存:
```bash
mkdir -p ${BASE}/detail ${BASE}/week ${BASE}/month

# detailは上書き
# week, monthは追記（既存ファイルの末尾に追加）
```

## 重要な注意事項

1. **単なるコミットログの羅列にしない**
   - 作業の背景、目的、ビジネス価値を記述する
   - 技術的な詳細だけでなく、なぜその作業が必要だったかを説明する

2. **時系列で整理**
   - コミット時刻順に作業内容を並べる
   - 関連する作業はまとめて記述する

3. **成果を明確に**
   - 機能追加、不具合修正、保守性向上などカテゴリ分けする
   - 具体的な内容や効果を含める

4. **次回への繋がり**
   - 未完了のタスクや次回作業予定を明記
   - ブランチ名を記録して作業の継続性を保つ

5. **日本語で出力**
   - すべて日本語で記述する
   - 絵文字は使用しない

## 引数の扱い

- **引数なし**: 昨日の日報を生成
- **引数あり（YYYYMMDD形式）**: 指定日の日報を生成（例: `20260226`）
- **引数あり（YYYY-MM-DD形式）**: 指定日の日報を生成（後方互換）

## gitコミットがない場合

gitコミットがない日でも、以下の情報から作業内容を推測して日報を生成:
- ファイル変更履歴（mtime基準）
- 会話履歴での議論や調査内容
- 設計検討やレビュー作業

例:
```markdown
### 1. 要件定義と設計検討 (10:00-12:00)

**作業内容**:
- 機能仕様の確認
- 既存実装との比較分析
- 実装方針の検討

**成果**:
- 仕様ドキュメント更新
- 実装方針の決定
```
