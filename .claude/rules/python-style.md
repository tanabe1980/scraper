# Python実装ルール(pdf-app)
- コメントに余計な空白文字を挟まない。
- コメントに絵文字は使用しない。
- PEP8に従う。ただし1行の文字数は150文字程度まで許容する。
- docstringはGoogle Style Python Docstringsに準拠する。
- ファイル先頭に看板(概要/影響)を付ける。
  - テンプレート
  ```python
  """
  ファイル名
  概要:
  影響:
  """
  ```
- クラス/関数/メソッドに看板(概要/入出力/データ入出力元)を付ける。
  - テンプレート
  ```python
  """
  名称:
  概要:
  input: 型: 説明
  output: 型: 説明
  データ入力: file|BQ|DB|Delta
  データ出力: file|BQ|DB|Delta
  """
  ```
- コードを新規作成または修正した場合、必ずテストを追加または修正する。必要に応じて設計ドキュメントも更新する。
