# データベース運用ルール

## DBリセット時のバックアップ必須
- `supabase db reset`やマイグレーション適用など、DBを初期化・リセットする操作を行う前に**必ずバックアップを取得**する。
- バックアップ保存先: `./temp/backup/`
- ファイル名形式: `backup_YYYYMMDD_HHmmss.json`
- バックアップ対象: 全テーブルデータ(auth.usersを含む)
- リセット後はバックアップからデータを復元可能な状態を維持する。

## バックアップ手順
1. リセット前にSupabaseのデータをJSON形式でエクスポート
2. `./temp/backup/`に保存
3. リセット実行
4. 必要に応じてバックアップからインポート
