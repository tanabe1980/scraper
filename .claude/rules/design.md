# 設計ドキュメント運用

## フォルダ構成

| フォルダ | 用途 |
|---|---|
| `docs/00_overview/` | 基本設計・意思決定記録(ADR) |
| `docs/01_design/` | 詳細設計・仕様書 |
| `docs/02_coding/` | プログラミング設計 |
| `docs/03_test/` | テスト設計 |
| `docs/04_ops/` | 実装・運用手順 |
| `docs/05_local/` | ローカル運用手順 |
| `docs/06_templates/` | 申告書テンプレート(年度別) |
| `docs/07_vendor-templates/` | ベンダーテンプレート |
| `docs/08_sample-data/` | インポート用サンプルデータ |

## 変更時のルール
- 変更が設計に影響する場合、`docs/01_design/`の該当ドキュメントを更新する。
- 設計判断を伴う変更の場合、`docs/00_overview/`にADRを追加する。
- コードを新規作成または修正した場合は、設計更新の要否を必ず判定する。
- 変更時は以下を最低限書く。
  - 目的/背景
  - 変更点
  - 影響範囲
  - テスト観点

```markdown
# ADR-NNN: タイトル

## 結果・影響

## 代替案
```

## ドキュメント配置の判断基準

| 変更内容 | 更新先 |
|---|---|
| 基本方針・PLデータ構造の変更 | `docs/00_overview/` |
| アーキテクチャ・技術選定の決定 | `docs/00_overview/` (ADR) |
| 機能仕様・マスタ仕様の変更 | `docs/01_design/` |
| プログラミング設計の変更 | `docs/02_coding/` |
| テスト設計の変更 | `docs/03_test/` |
| 操作手順・環境構築手順の変更 | `docs/04_ops/` |
| ローカル運用・手引き参照情報 | `docs/05_local/` |
